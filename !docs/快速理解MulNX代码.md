# 本文档旨在帮助任何对代码感兴趣的人，包括但不限于开发者，理解MulNX是如何工作的（当然只是简单介绍，具体请看代码）

- 会同时介绍随时间，MulNX干了什么，以及相关的代码说明
- **重要声明**：由于本项目涉及技术栈过于宽广，所以默认读者有相关知识储备，最起码有想要了解的模块对应的知识储备

## 注入阶段

- 当你双击运行注入器后，注入器开始等待，当你按下打开CS2后，它会检查cs2.exe是否已经运行
- 如果已经运行，那么它会拒绝下一步
- 如果没有运行，那么它会附加-insecure等参数打开CS2，这是为了关闭VAC反作弊

## 注入后

### 创建时

- 参看CS2OBTool/DllMain/DllMain.cpp的代码，可以发现，MulNX是一个模块化的架构，包括系统本身，绝大部分都只是一个平等的模块
- 可以看到CreateSystemModules()在 (*Core->ModuleManager())后立刻调用，这是必须的，因为框架需要工作
- 然后通过流式接口注册了模块，这也是添加功能的手段，即继承模块基类，书写功能，然后在这里注册上去
- 需要注意的是，100及更小的数字是MulNX保留字，任何模块创建，ID小于等于100，均是未定义行为

### 初始化

- 当核心启动，它加载的核心启动器将会驱动初始化进行，相关线程，钩子将被创建
- 随后依赖注入将被进行，让组件可以访问系统等
- 然后组件被启动，运行开始

### 运行时

- 框架驱动整个系统所有组件运转，并线程安全地处理消息，消息管理器会处理消息传递，让整个框架所有模块得以安全协作

### 关闭时

- 框架并没有做关闭功能，因为预期是其生命周期与整个游戏一样长，当游戏关闭，操作系统会自动回收资源
