# 构建指南

## 声明

- **构建原则**：最低构建难度
- **项目编码**: 完全UTF-8（CMake已设定utf-8编译选项）

## 开发环境系统要求

- **推荐使用Windows11**
- **网络要求**：我不知道你是怎么得到这个项目的，但是也许你并不会访问GitHub，实际上，大陆地区访问GitHub确实有难度。这里推荐使用微软应用商店自带的免费工具 Watt Toolkit，下载后打开，左侧选择网络加速，右边加速设置，选择Hosts模式，然后中间找到github，勾选所有项，然后上面启动“一键加速”。

## 项目结构验证

构建前必须确保**整个项目的相对目录结构**与仓库原始结构完全一致。

**重要说明**：

- 任何文件夹结构变更都可能导致编译失败
- 请勿移动、重命名或删除任何必要的文件夹

## 关于输出

- **声明**：本项目所有的python脚本均**不是必须的**！它只是帮助你快速打包而已，**完全可以**手动打包！！
- 所有生成的文件都会存放在源代码根目录的 bin 文件夹下，结构如下：

- bin/MulNX/ : 存放直接输出产物
- bin/output/ : 运行 pack.py 后合并过滤的文件（脚本需手动执行）
- bin/MulNX.zip : 运行 pack.py 后自动打包的压缩包

- calculatelines只是用来统计非第三方的代码行数的小工具，**不是项目必须！**
