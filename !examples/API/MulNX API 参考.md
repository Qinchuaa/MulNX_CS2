# MulNX API 参考文档

## 核心原理

- MulNX 内部通过**消息总线**来传递信息，因此，外部接口实际上是和消息总线交互的

## 连接方式

- 连接：WebSocket，默认 ws://127.0.0.1:55202

## 消息格式

- MulNX 的外部网关忠实地将网络消息直接解释为MulNX内部消息并投入消息总线，故消息必须严格遵守格式
- 消息格式：JSON 文本帧 {"type":"...", "p1":"...", "p2":"...", "str1":"...", "str2":"..."}
- p1/p2：必须为 Base64 编码的 8 字节小端二进制数据（double/uint64/两32位值等）
- 仅支持单向发送指令，无数据查询或事件订阅

## 消息类型

- 消息类型以MulNX的CPP代码为准，故推荐使用纯type（不包含参数字段）命令，直到相关消息稳定
- 具体支持可参考MulNX日志，会有订阅记录，然后可反查CPP部分如何理解数据，进而构造消息
